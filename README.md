# claude-vps

Автоматизированная установка Claude Code + веб-панели CloudCLI на чистый VPS (Ubuntu 24.04).

## Подготовить заранее

- Чистый сервер Ubuntu 24.04, SSH-доступ под root (пароль или ключ)
- Домен с A-записью, указывающей на IP сервера (проверить на [whatsmydns.net](https://www.whatsmydns.net))
- Подписка Claude Pro/Max

## Установка

Клонируйте репозиторий на сервер (под root, при первом входе):

```bash
apt update && apt install -y git   # если git ещё не установлен
git clone https://github.com/ваш-аккаунт/claude-vps.git
cd claude-vps
chmod +x *.sh
```

### Шаг 1 — сервер, пользователь, безопасность

```bash
./01-server-setup.sh
```

Скрипт спросит имя нового пользователя, перенесёт на него ваш SSH-ключ, **скопирует этот репозиторий в его домашнюю папку** (`~/claude-vps`), **остановится и попросит подтвердить в отдельном окне терминала**, что вход под новым пользователем работает — только после этого отключит пароль и root-логин. Дальше сам поставит fail2ban, базовый ufw (порт 22), Node.js и Claude Code (сам бинарник — без входа в аккаунт, вход будет позже через браузер) — причём Claude Code ставится в собственную npm-директорию нового пользователя (не через root), чтобы автообновления сразу работали без прав root.

### Шаг 2 — веб-панель

```bash
ssh новый-пользователь@IP-сервера
cd ~/claude-vps
./02-webpanel-setup.sh
```

Спросит домен, поставит CloudCLI, Caddy (авто-HTTPS), откроет 80/443 в ufw, дополнительно проверит и при необходимости доделает SSH-хардненинг из шага 1.

### Шаг 3 — открыть панель и войти в аккаунт Claude (всё через браузер)

Откройте `https://ваш-домен` (с любого устройства, VPN не нужен):

1. **Сразу зарегистрируйте владельца панели** — первый вошедший становится единственным владельцем.
2. Войдите в аккаунт Claude: **Настройки → Агенты → вкладка Claude → Аккаунт → «Войти снова»**. Откроется встроенный веб-терминал, ссылка авторизации там сразу кликабельна — просто нажмите на неё. Копировать ссылку руками, как раньше через SSH, не нужно — это убирает основную причину прошлых ошибок входа (`Invalid client_id`, `Redirect URI not supported` и т.п., связанных с порчей ссылки при копировании в узком терминале).

   **VPN всё равно нужен на этот шаг** — сама страница авторизации Anthropic по-прежнему недоступна с российского IP, кликабельность решает только проблему копирования, а не блокировку по геолокации: включите VPN на минуту, пройдите авторизацию, скопируйте код, выключите VPN, вставьте код обратно в веб-терминал.

### Шаг 4 — Telegram-бот (опционально)

Telegram-бот как быстрый интерфейс к Claude Code на том же VPS — вопросы, файлы и картинки прямо из чата, без захода по SSH. Полные бизнес-требования — в `telegram-bot-requirements.md`.

Подготовить заранее:

1. **Токен бота** — в Telegram написать [@BotFather](https://t.me/BotFather), `/newbot`, дать имя. Можно создать и позже — скрипт спросит и разрешит оставить пустым.
2. **Telegram user ID тех, кому бот будет отвечать** — написать [@userinfobot](https://t.me/userinfobot), он пришлёт ID в ответ. Можно указать несколько через запятую — остальным бот молчит. У каждого разрешённого пользователя — свой отдельный контекст сессии (не общий).

Запуск (под тем же пользователем, что и шаг 2, на сервере, где уже стоит `claude`):

```bash
cd ~/claude-vps
./03-telegram-bot-setup.sh
```

Скрипт создаст изолированную директорию для данных бота (`~/telegram-bot-data` — отдельно от `~/.ssh` и прочих конфигов), спросит токен и user ID, поставит зависимости (`TelegramBot/`, Node.js), включит systemd-службу `telegram-bot` и systemd-таймер `telegram-bot-cleanup` (ежедневная чистка временных файлов).

Что делает бот:
- текст → `claude -p` → ответ в чат; файлы и картинки принимает, сохраняет в изолированную папку и передаёт путь Claude Code для анализа
- сессия продолжается между сообщениями, `/new` сбрасывает контекст
- инструменты Claude Code внутри сессий бота ограничены чтением (`Read`, `Grep`, `Glob`) + веб-поиском — Bash и запись файлов недоступны

Если токен или user ID не заполнили сразу — впишите их в `TelegramBot/.env` (`ALLOWED_USER_IDS=111111,222222`) и запустите `sudo systemctl enable --now telegram-bot`. Чтобы добавить ещё одного пользователя позже — допишите его ID через запятую в `ALLOWED_USER_IDS` и перезапустите: `sudo systemctl restart telegram-bot`.

Поменять порог свободного места (по умолчанию 1 ГБ) или срок хранения временных файлов (по умолчанию 72 часа) — отредактировать `MIN_FREE_SPACE_MB` / `TEMP_FILE_TTL_HOURS` в `TelegramBot/.env` и перезапустить службу: `sudo systemctl restart telegram-bot`.

Логи: `sudo journalctl -u telegram-bot -f` (сам бот) и `sudo journalctl -u telegram-bot-cleanup -f` (очистка).

## Что не хранится в этом репозитории (вводится вручную)

- Данные аккаунта Anthropic (вводятся через браузер на шаге 3)
- Логин и пароль веб-панели CloudCLI
- Пароль нового пользователя на сервере
- Токен Telegram-бота и разрешённый user ID (`TelegramBot/.env`, шаг 4)

## Восстановление на новом сервере

Повторить шаги «Установка» на новом VPS. Старый сервер трогать не нужно.

## Безопасность

- Оба скрипта идемпотентны — повторный запуск не должен ничего сломать.
- `01-server-setup.sh` требует root и явного подтверждения доступа перед отключением пароля/root-логина.
- `02-webpanel-setup.sh` требует НЕ root (обычный пользователь с sudo) и наличия установленного `claude` (ставится в шаге 1); вход в аккаунт не обязателен на этом этапе.
- `03-telegram-bot-setup.sh` тоже требует НЕ root. Данные бота (входящие файлы, сессии, лог) хранятся в отдельной от `~/.ssh`/конфигов директории `~/telegram-bot-data`. Внутри сессий бота Claude Code ограничен инструментами чтения — Bash и запись файлов недоступны, бот не может ничего менять или выполнять на сервере.
- Секреты в репозиторий не попадают — только структура и логика установки.
