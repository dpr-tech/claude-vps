# VPS Claude Setup

Автоматизированная установка Claude Code + веб-панели CloudCLI на чистый VPS (Ubuntu 24.04).

## Подготовить заранее

- Чистый сервер Ubuntu 24.04, SSH-доступ под root (пароль или ключ)
- Домен с A-записью, указывающей на IP сервера (проверить на [whatsmydns.net](https://www.whatsmydns.net))
- Подписка Claude Pro/Max
- Разовый доступ к VPN на своём устройстве (для шага авторизации, см. ниже)

## Установка

Клонируйте репозиторий на сервер (под root, при первом входе):

```bash
apt update && apt install -y git   # если git ещё не установлен
git clone <ссылка-на-этот-репозиторий>
cd vps-claude-setup
chmod +x scripts/*.sh
```

### Шаг 1 — сервер, пользователь, безопасность

```bash
./scripts/01-server-setup.sh
```

Скрипт спросит имя нового пользователя, перенесёт на него ваш SSH-ключ, **остановится и попросит подтвердить в отдельном окне терминала**, что вход под новым пользователем работает — только после этого отключит пароль и root-логин. Дальше сам поставит fail2ban, базовый ufw (порт 22), Node.js и Claude Code.

### Шаг 2 — вход в Claude (вручную, не автоматизируется)

```bash
ssh новый-пользователь@IP-сервера
claude
```

Выберите **Claude account**. Ссылку авторизации нужно открыть с не-российского IP:
1. Скопируйте ссылку из терминала.
2. Включите VPN на своём устройстве (разово, на минуту).
3. Откройте ссылку, войдите, скопируйте код.
4. Выключите VPN.
5. Вставьте код в терминал сервера.

### Шаг 3 — веб-панель

```bash
cd vps-claude-setup
./scripts/02-webpanel-setup.sh
```

Спросит домен, поставит CloudCLI, Caddy (авто-HTTPS), откроет 80/443 в ufw, дополнительно проверит и при необходимости доделает SSH-хардненинг из шага 1.

### Шаг 4 — открыть панель

Откройте `https://ваш-домен` в браузере (с любого устройства, VPN не нужен) и **сразу** зарегистрируйте владельца панели — первый вошедший становится единственным владельцем.

## Что не хранится в этом репозитории (вводится вручную)

- API-ключ / данные аккаунта Anthropic
- Логин и пароль веб-панели CloudCLI
- Пароль нового пользователя на сервере

## Восстановление на новом сервере

Повторить шаги «Установка» на новом VPS. Старый сервер трогать не нужно.

## Безопасность

- Оба скрипта идемпотентны — повторный запуск не должен ничего сломать.
- `01-server-setup.sh` требует root и явного подтверждения доступа перед отключением пароля/root-логина.
- `02-webpanel-setup.sh` требует НЕ root (обычный пользователь с sudo) и наличия `~/.claude` (то есть уже пройденного `claude login`).
- Секреты в репозиторий не попадают — только структура и логика установки.
